What it does

Everything you need. Nothing you don't.

◆

AI Meta Generator

Auto-writes a 150–160 character, SEO-optimized meta description the moment a post is published. Custom prompt with {title}, {content}, {excerpt}, {language} placeholders. Fallback extraction works without any API key.

◆

Bulk Generator

Batch-process your entire back-catalogue. Finds all published posts with no meta description and generates them in configurable batches. Live progress log. Running cost estimate. Rate-limited, retry logic, transient lock.

◆

Schema.org (GEO)

Injects JSON-LD structured data for AI and search engines — Organization, Article, Author, Speakable, BreadcrumbList. Auto-suppresses duplicates when Rank Math or Yoast is active.

◆

GEO Quick Overview

AI-generated per-post content block with a short summary, key bullet points, and optional FAQ — rendered as a collapsible <details> element. Helps AI retrieval systems extract structured answers directly from your content. Auto-generates on publish, fully editable in the post editor.

◆

llms.txt

Serves a machine-readable content index at /llms.txt for AI systems. Follows the emerging llms.txt standard. ETag / Last-Modified HTTP caching, pagination for large sites, custom sections.

◆

robots.txt Manager

Block 13 known AI training and data-harvesting crawlers individually from the WordPress admin. GPTBot, ClaudeBot, Google-Extended, PerplexityBot, Bytespider, and more. No manual file editing.

◆

Crawler Log

See which AI bots are visiting your site. Stores bot name, hashed IP (SHA-256, never plain text), and requested URL. 90-day auto-purge via WP-Cron. 30-day summary on the dashboard.

Your keys.
Your control.

API keys are obfuscated before database storage using XOR encryption with a key derived from your WordPress authentication salts. They never appear in plain text in database dumps or export files.

The admin UI always shows keys masked — only the last 5 characters visible.

For maximum security: define keys in wp-config.php. The plugin uses them automatically — nothing stored in the database at all.

// Stored format in DB
bre1:<base64(xor(key, sha256(AUTH_KEY + SALT)))>

// Admin UI — always masked

// wp-config.php alternative (recommended)
define('BRE_OPENAI_KEY',    'sk-...');
define('BRE_ANTHROPIC_KEY', 'sk-ant-...');
define('BRE_GEMINI_KEY',    'AI...');
define('BRE_GROK_KEY',      'xai-...');
